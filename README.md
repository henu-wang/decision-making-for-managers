# Decision Making for Managers

> A practical guide for managers to make faster, better decisions — and help their teams do the same.

[![KeepRule](https://img.shields.io/badge/Explore_Scenarios-KeepRule-blue)](https://keeprule.com/en/scenarios)

## The Manager's Decision Dilemma

Managers face a unique challenge: they must make **their own decisions well** while also **enabling their team to decide without them**. The best managers are not the best deciders — they build the best decision-making systems.

## The Decision Authority Matrix

| Decision Type | Who Decides | Manager's Role |
|--------------|-------------|---------------|
| **Type 1: Irreversible** | Manager + team input | Decide carefully, own the outcome |
| **Type 2: Reversible** | Team member | Delegate, don't micromanage |
| **Type 3: Routine** | Whoever is closest | Set guidelines, get out of the way |
| **Type 4: Strategic** | Leadership team | Facilitate alignment, build consensus |

*Inspired by Jeff Bezos: "Most decisions are Type 2 and should be made quickly."*

## The RAPID Framework

For complex organizational decisions:

| Role | Responsibility |
|------|---------------|
| **R**ecommend | Proposes the decision and gathers input |
| **A**gree | Must agree for the decision to proceed (has veto) |
| **P**erform | Executes the decision once made |
| **I**nput | Provides information and perspective |
| **D**ecide | The single person who makes the final call |

**Key rule:** There must be exactly **one D** for every decision.

## Speed vs. Quality Matrix

```
                    High Quality
                        │
     Analysis           │          Sweet Spot
     Paralysis          │          (aim here)
                        │
   ──────────────────────┼──────────────────── Fast
                        │
     Worst Case         │          "Move fast
     (avoid)            │          and break things"
                        │
                    Low Quality
```

### Guidelines:
- **Reversible decisions:** Optimize for speed (70% confidence is enough)
- **Irreversible decisions:** Optimize for quality (require 90% confidence)
- **Never optimize for:** Consensus (it's a trap)

## Common Manager Decision Traps

### 1. The Approval Bottleneck
- **Symptom:** Team waits for your approval on everything
- **Fix:** Define clear decision boundaries; tell team "decide without me if X"

### 2. The HIPPO Effect
- **Symptom:** Highest-Paid Person's Opinion always wins
- **Fix:** Share your opinion last; ask for data first

### 3. The Meeting Trap
- **Symptom:** Every decision requires a meeting
- **Fix:** Use async decision docs; meetings only for debate, not updates

### 4. The Perfectionism Trap
- **Symptom:** Waiting for complete information before deciding
- **Fix:** Ask "What would we need to know to change this decision?" — often very little

## The Manager's Decision Template

```markdown
## Decision: [Title]
**Deadline:** [Date]
**Decider:** [Name]

### Context
What problem are we solving? Why now?

### Options
1. Option A — [brief description]
2. Option B — [brief description]
3. Do nothing — [what happens]

### Recommendation
[Which option and why]

### Risks
[What could go wrong and mitigation]

### Decision
[Final choice, recorded after discussion]
```

## Delegation Decision Tree

```
Can someone on my team do this?
├── YES → Will it develop them?
│   ├── YES → Delegate (even if they'll do it differently)
│   └── NO → Delegate anyway (free up your time)
└── NO → Can they learn?
    ├── YES → Delegate with coaching
    └── NO → Do it yourself (rare)
```
